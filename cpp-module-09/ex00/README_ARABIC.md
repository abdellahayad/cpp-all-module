🔐 التحقق من الصحة (Validation)
════════════════════════════════════════════════════════════════════════════

يتم فحص البيانات على عدة مستويات:

1️⃣  فحص الملف:
    ├─ هل الملف موجود وقابل للفتح؟
    └─ → خطأ إن لم يكن كذلك

2️⃣  فحص الصيغة:
    ├─ هل يوجد الفاصل " | "؟
    └─ → Error: bad input

3️⃣  فحص التاريخ:
    ├─ الطول = 10 أحرف؟
    ├─ الفواصل في الموضع الصحيح (4, 7)؟
    ├─ باقي الأحرف أرقام فقط؟
    ├─ السنة 2009-2024؟
    ├─ الشهر 1-12؟
    ├─ اليوم 1-31؟
    └─ → Error: invalid input

4️⃣  فحص القيمة:
    ├─ هل هي رقم صحيح؟
    ├─ هل هي موجبة (≥ 0)؟
    ├─ هل لا تتجاوز 1000؟
    └─ → Error: not a positive number / too large

5️⃣  فحص البيانات التاريخية:
    ├─ هل يوجد سعر للتاريخ؟
    └─ → Error: no rate found

🧮 خطوات الحسابة بالتفصيل
════════════════════════════════════════════════════════════════════════════

إذا كان المدخل:
    2011-06-15 | 0.5

الخطوات:

Step 1: تحليل
    ├─ date = "2011-06-15"
    ├─ value = "0.5"
    └─ بعد التنظيف (حذف المسافات):
       date = "2011-06-15"
       value = "0.5"

Step 2: التحقق
    ├─ isValidDate("2011-06-15") → true ✓
    └─ isValideValue("0.5") → true ✓

Step 3: البحث عن السعر
    البحث في الخريطة:
    
    a) البحث بدقة عن "2011-06-15"
       → قد لا يوجد
    
    b) lower_bound("2011-06-15")
       → يجد أول تاريخ >= "2011-06-15"
       → قد يكون "2011-06-20" مثلاً
    
    c) الانتقال للسابق (--it)
       → "2011-06-15" أو أقدم
    
    d) إرجاع السعر
       → rate = 24.88

Step 4: الحسابة
    result = amount × rate
           = 0.5 × 24.88
           = 12.44

Step 5: الطباعة
    2011-06-15 => 0.50 = 12.44
                         ↑ عدد عشري (2 منزلة)

📈 تعقيد الخوارزمية (Time Complexity)
════════════════════════════════════════════════════════════════════════════

العملية                    التعقيد
─────────────────────────  ─────────
تحميل البيانات (loadData)  O(n log n)  ← n = عدد الأسعار
                           (بسبب إدراج في map)

معالجة سطر واحد:
├─ تحليل النص              O(m)        ← m = طول السطر
├─ التحقق من التاريخ        O(1)
├─ التحقق من القيمة         O(1)
├─ البحث عن السعر            O(log n)    ← map::find
└─ الحسابة والطباعة          O(1)

المجموع لـ k أسطر:         O(n log n + k(m + log n))

مثال الأداء:
  ← تحميل 1614 سعر: سريع جداً (~0.001 ثانية)
  ← معالجة 8 أسطر: سريع جداً (~0.001 ثانية)
  ← المجموع: فوري تقريباً
